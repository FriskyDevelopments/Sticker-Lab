# Brand Architecture

## Overview
The ecosystem works best when each brand has a clear job.

Without that separation, the parent studio starts to compete with the product, the product starts to absorb identity language, and the overall system feels improvised. The goal of this architecture is to keep the ecosystem legible while preserving the distinct energy of each layer.

## The three-layer model
### 1. Frisky Developments
**Role:** parent creative studio / creator / signature / giver

Frisky Developments is the originating presence behind the work. It provides authorship, warmth, and personality. It should frame the ecosystem, not overpower it.

### 2. STIX MΛGIC
**Role:** engine / magic / expressive system

STIX MΛGIC is the core product/system expression. It is where transformation, animation, aura, and magical responsiveness live. When people encounter the active system, STIX should usually be the primary product brand.

### 3. LORE
**Role:** identity layer / profile layer / user-facing presence

LORE is the layer that turns generated or expressed magic into identity. It should carry the language of profile, memory, continuity, and presence.

## Narrative relationship
The simplest way to explain the ecosystem is:
- **Frisky gives the magic**
- **STIX is the magic**
- **LORE gives that magic identity**

That line should be treated as a strategic shorthand, not a slogan that must appear everywhere verbatim.

## Decision rules
Use **Frisky Developments** when the moment needs:
- authorship
- studio context
- ecosystem framing
- creative intent
- a human signature behind the tools

Use **STIX MΛGIC** when the moment needs:
- product emphasis
- system behavior
- transformation or animation language
- experience-level brand presence

Use **LORE** when the moment needs:
- identity framing
- user profile/presence language
- persistence, memory, or expression tied to a person

## Naming hierarchy in practice
Examples:
- “Frisky Developments is the creative studio behind STIX MΛGIC and LORE.”
- “STIX MΛGIC powers the expressive system.”
- “LORE gives each expression an identity layer.”

This keeps the hierarchy clean:
- parent brand first when introducing the ecosystem
- product/system brand first inside the actual experience
- identity brand first where user presence is the focus

## What this architecture prevents
A clear brand architecture helps avoid:
- treating Frisky like a generic corporate umbrella
- making STIX carry every narrative burden in the ecosystem
- collapsing LORE into a feature instead of a distinct layer
- inconsistent descriptions across documentation or future interfaces

## Documentation guidance
Whenever new docs are added, collaborators should check:
1. Which layer is actually being described?
2. Is the language consistent with that layer’s role?
3. Is Frisky being used as authorship rather than as the product itself?
4. Is STIX still the hero when discussing the active system?
5. Is LORE still reserved for identity and presence?

A coherent ecosystem will come less from slogans and more from repeated role discipline.
